Template Anatomy

The dining-events template is a single, clean composition designed to survive Mailchimp inlining and Outlook rendering while still feeling editorial. It uses strong hierarchy, quiet spacers, and one clear call-to-action rhythm.

Structural Pattern

Header: dark blue bar with centered shield mark.
Intro: warm tan section with faint mountain drawing and a single headline.
Content: stacked white event cards, each with title, time, date badge, image, copy, and tan CTA.
Closing: warm tan concierge section with primary CTA and no separate heavy footer block.
Legal: unsubscribe and address lines sit quietly within the closing treatment.

Brand Tokens in Email

Dark Blue: #1e2d3a for structure, badges, and primary buttons.
Warm Tan: #ede9e1 for intro and closing blocks, plus softer CTA moments.
Headlines: Cormorant Garamond light weight only.
Body/UI: Lato 300 / 400 / 700.
Imagery: Mailchimp CDN assets only. Do not rely on local file paths.

Non-Negotiable Implementation Rules

Email is the part of the package with the least flexibility. These are the rules that keep the templates editable in Mailchimp and stable across Gmail, Apple Mail, and Outlook.

Layout

Use tables only

Do not introduce flexbox, grid, or positioning-based layout. Outlook's rendering engine will not behave reliably with those choices.

Editability

Keep mc:edit clean

Every mc:edit value must be unique, and the attribute belongs on the parent <td>, not on image or anchor tags.

Compliance

Preserve merge tags

Do not remove required Mailchimp merge tags such as *|UNSUB|* and *|LIST:ADDRESSLINE|*.

Compatibility

Avoid modern CSS shortcuts

Keep styles conservative and inline-friendly. Outlook, Gmail clipping, and mobile overrides all punish unnecessary complexity.

Assets

Use the Mailchimp CDN

Source logos and drawings from the existing mcusercontent.com URLs. Local repo paths will not resolve in delivered campaigns.

Buttons

Keep bulletproof buttons

Buttons must use the table-cell pattern rather than styling a naked anchor, especially when Outlook support matters.

Working Checklist

Use this checklist whenever a new email template or campaign variation is being built from the Hoback system.

Build Checklist

Start from the existing dining-events structure when possible.
Keep the 600px centered container and responsive 620px breakpoint.
Use Mailchimp CDN image URLs for shield logo, mountain drawing, and building drawing.
Retain the hidden preview text directly after <body>.
Keep HTML size well under Gmail clipping thresholds.

QA Checklist

Verify Mailchimp does not strip required attributes after upload.
Send test emails to Gmail, Apple Mail, and Outlook.
Check that mobile stacking behaves correctly below 620px.
Confirm legal text, unsubscribe, and address tags are present.
Ensure no local repo paths remain inside the production template.
Recommended working path:
1. Read Hoback_MailChimp/CLAUDE.md
2. Review the relevant docs in Hoback_MailChimp/docs/
3. Start from Hoback_MailChimp/templates/dining-events.html
4. Test in Mailchimp and in real inboxes before calling the template done
