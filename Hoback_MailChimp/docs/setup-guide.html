Step 1 — Upload Images to Content Studio

Before importing the template, upload all images so Mailchimp hosts them and you have permanent URLs.

1

Log in to Mailchimp. In the left sidebar, go to Content → My Files (Content Studio).

2

Click Upload and select these 4 files from this folder:

  • logo-footer.svg
  • logo-nav.svg
  • tetons_drawing.png
  • hoback_drawing.png
3

After upload, click each image and copy its URL (click the image, then "Copy URL" or right-click and copy image address). Save these URLs — you'll need them in Step 2.

Note on SVG: Some email clients don't render SVG. If the logo doesn't display in test sends, export the SVGs as PNG files (at 2x resolution for retina) and upload those instead.

Step 2 — Update Image URLs in the Template

Open hoback-email-template.html in a text editor (TextEdit, VS Code, Notepad — anything works).

Find and replace these placeholder src values with the Mailchimp Content Studio URLs you copied:

src="logo-footer.svg"
Replace with your Mailchimp-hosted logo URL (appears 2 times — header and footer)
src="tetons_drawing.png"
Replace with your Mailchimp-hosted tetons drawing URL (appears 1 time — intro section)
src="hoback_drawing.png"
Replace with your Mailchimp-hosted hoback drawing URL (appears 1 time — closing section)

Tip: Use Find & Replace (Ctrl+H / Cmd+H) to swap all instances at once. The event placeholder images (hobackclub.com/images/dynamic/...) can stay as-is — the dining room manager will replace them in the editor.

Step 3 — Import the Template into Mailchimp

1

In Mailchimp, go to Content → Email templates.

2

Click Create Template.

3

Choose Code your own → Paste in code.

4

Open your updated hoback-email-template.html, select all the code (Ctrl+A / Cmd+A), copy it, and paste into the Mailchimp code editor.

5

Click Preview to verify the layout looks correct — check the drawing, logos, date badges, and footer.

6

Name the template (e.g., "Hoback Dining Events - 3 Event") and click Save.

Template size limit: Mailchimp allows up to 200KB of HTML. This template is well under that. However, Gmail clips emails larger than 102KB — keep total content (including images loaded inline) lean.

Step 4 — Creating a Campaign from the Template

This is the workflow the dining room manager will follow each time they send a dining events newsletter:

1

Go to Create → Email (Regular email).

2

Set the audience (who receives the email), subject line, and sender.

3

Under "Content", click Design Email and choose Saved Templates. Select "Hoback Dining Events - 3 Event".

4

The visual editor opens with all the editable sections. Click on any content block to edit it:

What the Manager Can Edit

  • Event details (mc:edit="event_1_details", etc.) — Change the time, title, and date badge number/month
  • Event images (mc:edit="event_1_image", etc.) — Click the image, then "Replace" to pick a new photo from Content Studio or upload one
  • Event descriptions (mc:edit="event_1_cta", etc.) — Edit the description text, price, and guest limit. The "Reserve Your Seat" button link can be changed by clicking the button text and editing the URL
  • Intro section — Update "The Great Hall" subheading or headline if needed
  • Closing section — Update concierge phone number or "View Full Menu" link
  • Footer — Links and contact info (usually stays the same)
5

Click Preview and Test → Send a Test Email to yourself before sending to members.

6

Send or Schedule the campaign.

Pro tip: After the first campaign, the manager can go to Campaigns, find the previous dining email, and click Replicate. This copies everything — they just update the 3 events instead of starting from the template each time.

Image Specifications for the Dining Manager

When uploading event photos to Content Studio:

  • Size: 1200 x 600 pixels (2x for retina — displays at 600 x 300)
  • Format: JPG or PNG (JPG preferred for photos — smaller file size)
  • Max file size: Under 1MB per image for fast loading
  • Orientation: Landscape (horizontal) — portrait images will be cropped
  • Content: Food, table settings, the dining room, chef — anything that sells the event
  • Always add alt text when replacing images in the editor (e.g., "Wine Pairing Dinner at The Great Hall")

Required Merge Tags (Already Included)

The template already includes all legally required Mailchimp merge tags. Don't remove these:

  • *|UNSUB|* — Unsubscribe link (required by CAN-SPAM)
  • *|LIST:ADDRESSLINE|* — Your physical mailing address (required by CAN-SPAM)
  • *|UPDATE_PROFILE|* — Link for members to update their email preferences
  • *|CURRENT_YEAR|* — Auto-fills the current year in the copyright line

Make sure your Mailchimp audience settings have the correct physical address filled in (Settings → Audience name and defaults → Contact information). This is what *|LIST:ADDRESSLINE|* pulls from.

Testing Checklist

Before the first real send, verify these:

  1. Send a test email from Mailchimp to yourself (Gmail, Apple Mail, and Outlook if possible)
  2. Check that logos display — if SVGs don't show, replace with PNG versions
  3. Check that event images load and aren't stretched or cropped badly
  4. Click every link and button — "Reserve Your Seat", "View Full Menu", phone number, social links
  5. Verify the unsubscribe link works
  6. Check the email on a phone — the layout should stack to single column
  7. Verify the physical address appears in the footer (from your Mailchimp audience settings)
  8. Check that the preview text shows in your inbox list (the "This month at The Great Hall..." line)

Troubleshooting

Images not showing

Make sure image URLs point to Mailchimp's CDN (they should start with https://mcusercontent.com/... or https://gallery.mailchimp.com/...). Local file paths like logo-footer.svg won't work in sent emails.

Logo not rendering in Outlook

Outlook desktop doesn't support SVG. Export the logo as a PNG (at least 140px wide for the header, 100px for the footer) and re-upload to Content Studio.

Fonts look different in some email clients

Google Fonts (Cormorant Garamond, Lato) render in Apple Mail, iOS Mail, and the Gmail app. In Outlook and some webmail clients, they fall back to Georgia (for headlines) and Helvetica/Arial (for body text). This is expected and intentional — the fallback fonts are specified in the template.

Email looks broken in Outlook

The template includes Outlook-specific conditional comments (<!--[if mso]>) for the date badges and closing section layout. If something still looks off in Outlook, the issue is likely a missing closing tag. Validate your HTML before pasting into Mailchimp.

Mailchimp says "merge tag not found"

This usually means an HTML tag is unclosed somewhere in the template, which prevents Mailchimp from parsing the merge tags. Check for any unclosed <td>, <tr>, or <table> tags.
